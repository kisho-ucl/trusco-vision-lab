# Hands-on Investigation Plan

## 目的

映像ベース作業認識について、文献調査だけで研究課題を決めるのではなく、実際に既存手法を触りながら、物流倉庫の俯瞰映像に対して何ができて何が難しいかを確認する。

特に、以前の査読対応で手法の新規性や既存研究との差分を問われた経験を踏まえ、既存技術を適用すれば解ける部分と、研究課題として残る部分を実証的に切り分ける。

## 基本方針

いきなり大きなモデルを作らない。
まずは既存手法を組み合わせたbaselineを作り、どこで破綻するかを見る。

確認する順番:

1. 人物検出・追跡がどの程度できるか。
2. 得られたtracklet / trajectoryが作業認識に使えるか。
3. RGB特徴、pose特徴、zone / object contextを追加する必要があるか。
4. 少量ラベル条件でどこまで認識できるか。

## Step 1: Data Check

### 目的

共同研究先の映像が、研究・実験に使える条件を満たしているか確認する。

### 確認項目

- カメラ視点: 俯瞰、斜め俯瞰、水平に近い視点のどれか。
- 人数: 同時に何人程度映るか。
- 解像度: 人物cropや手元がどの程度見えるか。
- フレームレート: 行動認識に十分か。
- 画角: 作業者が画角外へ出る頻度。
- 遮蔽: 棚、台車、他作業者による遮蔽。
- 作業ラベル: 検品・仕分け・搬送が映像から判別可能か。
- プライバシー: 顔、名札、商品情報などの扱い。

### 成果物

- 映像サンプルの観察メモ
- 代表シーンのスクリーンショット
- 実験に使える/使えない条件の整理

## Step 2: Tracking Baseline

### 目的

既存の人物追跡手法で、倉庫映像から作業者IDと軌跡をどの程度安定して取得できるか確認する。

### 候補手法

- YOLO系 detector + ByteTrack
- YOLO系 detector + BoT-SORT
- YOLO系 detector + OC-SORT

### 見るポイント

- 人物検出漏れ
- ID switch
- 作業者同士の交差
- 台車や棚による遮蔽
- 画角外退出・再入場
- 作業服が似ている場合のID維持

### 成果物

- tracking結果の可視化動画
- 失敗例リスト
- 既存trackerで十分な部分と不十分な部分の整理

## Step 3: Tracklet / Trajectory Feature Baseline

### 目的

人物軌跡だけで、どの程度作業を分類できるか確認する。

### 入力特徴

- 位置
- 速度
- 加速度
- 移動距離
- 停止時間
- 作業エリア滞在時間
- エリア遷移
- 台車・棚・検品台などのzone proximity

### 想定できるラベル

- Transport: 移動距離や速度、台車付近の移動で比較的捉えやすい可能性。
- Inspect: 検品台付近の滞在で一部捉えられる可能性。
- Sort: 仕分けエリアでの滞在や往復で一部捉えられる可能性。

### 見るポイント

- trajectoryだけでTransportとInspect/Sortが分かれるか。
- InspectとSortがtrajectoryだけで区別できるか。
- zone情報を入れると改善するか。

### 成果物

- simple classifierの結果
- confusion matrix
- trajectoryだけで足りない作業の整理

## Step 4: RGB / Pose Feature Check

### 目的

trajectoryだけで足りない場合に、RGB特徴やpose特徴が必要かを確認する。

### RGB特徴の候補

- 人物crop特徴
- tracklet単位のvideo feature
- 作業者周辺cropのfeature

### Pose特徴の候補

- 2D pose keypoints
- skeleton sequence
- pose heatmap

### 見るポイント

- 俯瞰映像でpose推定が安定するか。
- 人物cropに作業の違いが残っているか。
- 手元や商品が小さすぎないか。
- RGB特徴が背景や場所に過適合しないか。

### 成果物

- pose推定結果の可視化
- RGB cropの観察メモ
- trajectoryに追加する価値がある特徴の判断

## Step 5: Low-annotation Setting

### 目的

フレーム単位の密なラベルではなく、少ないアノテーションで作業認識できるかを確認する。

### ラベル候補

- 作業区間ラベル
- 疎な時刻ラベル
- video-level label
- 既存作業記録やIMUからの弱教師

### 比較したい条件

- 全ラベル
- ラベル量50%
- ラベル量25%
- ラベル量10%
- 作業区間ラベルのみ
- 疎な時刻ラベルのみ

### 成果物

- ラベル量削減実験の設計
- 少量ラベルで崩れる条件の整理

## Step 6: Decide Thesis Scope

### 判断基準

以下を満たす方向を修論のVision側テーマにする。

- 10月までに実験できる。
- 自前映像で評価可能。
- 既存技術との差分を説明できる。
- IMU研究の「少量ラベル下の作業認識」と接続できる。
- 失敗例も研究的に意味を持つ。

### 有望な方向

第一候補:

> Tracklet / trajectoryを基盤とした、少量ラベル下の倉庫作業認識

第二候補:

> trajectory + zone contextによる作業区間認識

第三候補:

> IMUを弱教師として用いる映像ベース作業認識
