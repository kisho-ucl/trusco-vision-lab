# 2026-05-22

## Context

5月の進捗報告が終了した。報告では、Vision-based warehouse task recognitionの調査結果と、TRUSCO top-view tracking基盤から作業ラベル推定へ接続する方向を共有した。

## Feedback

教授から、以下の重要な示唆を得た。

### 1. TRUSCO映像処理基盤のSkill化

YOLO、SAM、カメラキャリブレーション、スティッチング、multi-camera trackingなどの基盤技術を、再利用可能な部品として整理しておくとよい。

それらを「Skill」としてAIに渡せる形にしておけば、特定の作業認識だけでなく、さまざまな映像処理・倉庫分析タスクに応用できる可能性がある。

示唆:

- 研究を単一モデルの提案に閉じず、TRUSCO映像処理基盤を使った研究開発プラットフォームとして捉えられる。
- 既存のキャリブレーション、スティッチング、trackingなどを、実験や応用に使いやすいAPI / workflow / documentationとして整理する価値がある。
- 将来的には、AIが複数の基盤技術を組み合わせて、目的に応じた分析pipelineを構築する方向も考えられる。

### 2. 時空間的な環境状態変化の利用

人の動きだけでなく、環境側の状態変化も作業認識に使える可能性がある。

例:

> 作業者の目の前にある物体が、ある時刻では閉じていて、別の時刻では開いている。

このような変化を、CLIPなどの画像特徴やvision-language modelで抽出できれば、作業内容を推定するための有用な情報になる可能性がある。

示唆:

- 作業認識を person-centric に限定せず、scene / object state transition も特徴として扱う。
- 「人がどう動いたか」だけでなく、「人の周辺環境がどう変化したか」を見る。
- 検品、仕分け、搬送の区別に、object stateやzone stateの時系列変化が効く可能性がある。

### 3. 半自動アノテーション基盤

現状では、約1年前に付与した少量ラベルを繰り返し使っている。今後Vision側の研究を進めるには、綺麗な行動ラベル付きデータを増やす仕組みが必要。

考えられる方向:

- tracking結果から、人ごとの時系列clipを自動抽出する。
- 抽出したclipをUI上で提示する。
- 人間は候補ラベルをマウスで選ぶだけでannotationできる。
- annotation済みclipが蓄積され、学習データ・評価データとして使える。
- 特徴量抽出やクラスタリングを使って、似たclipをまとめて提示し、ラベリング効率を上げる。

示唆:

- まず認識モデルを作る前に、データを増やすための半自動annotation workflowを作る価値がある。
- 実際に自分でラベリングしていく過程で、どの作業が紛らわしいか、どの特徴が効きそうかを発見できる可能性がある。
- ラベリング基盤自体も、TRUSCO映像処理基盤の重要なSkill / moduleになりうる。

### 4. Skill / module化による引き継ぎ性

このプロジェクト自体を育てていけば、今後の引き継ぎもかなり容易になりそう。

理想像:

- 新しく参加した人がgit cloneする。
- READMEやdashboardを読むと、研究の概要と現在地が分かる。
- surveyやlogsから、過去の調査・判断・失敗が追える。
- 映像処理の実装や手順がSkill / moduleとして整理されている。
- AIにプロジェクト内のSkillを渡すことで、基盤技術を組み合わせた新しい分析pipelineを組みやすくなる。

これは修論のためだけでなく、研究室内の継続的な研究資産としても価値がある。

## Updated Research Direction

現時点では、以下の4方向を今後の検討に加える。

1. TRUSCO映像処理基盤のSkill / module化
2. worker trajectoryに加えて、object / scene state transitionを使った作業認識
3. tracking結果からclipを抽出してラベルを付ける半自動annotation workflow
4. project knowledge / tools / skillsをgit管理し、引き継ぎや再利用を容易にする研究資産化

## Next

- TRUSCO映像処理基盤を、どのようなSkill / moduleとして整理できるか棚卸しする。
- YOLO、SAM、calibration、stitching、tracking、zone mapping、object state detectionなどをpipeline部品として整理する。
- object / scene state transitionを作業認識に使う具体例を考える。
- 既存のbbox crop分類やtrajectory分類に加えて、environment change featureを実験候補に入れる。
- 半自動annotation toolの最小構成を考える。
- clip extraction、label UI、feature extraction、clustering、dataset exportをannotation workflowの部品として整理する。

半自動annotation workflowのv0を設計・実装した。

追加したもの:

- `tools/annotation/index.html`
- `tools/annotation/app.js`

- `tools/annotation/sample_manifest.json`

v0の範囲:

- clip manifestを読み込む。
- clip metadata / zone hint / track summaryを表示する。
- Inspect / Sort / Transport / Other / Unclearを付与する。
- confidenceとnoteを残す。
- annotation結果をJSONLでexportする。
- localStorageに途中結果を保存する。

動作確認:

- sample manifestを読み込める。
- 1件目にInspectラベルとnoteを保存し、queueとprogressが更新される。
- desktop / mobile幅で表示が崩れないことを確認した。

次は、実際のtracking output形式を確認し、worker IDごとのclip manifestを生成するclip extraction scriptにつなぐ。
