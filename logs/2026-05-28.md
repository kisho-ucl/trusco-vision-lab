## 実装・修正

### annotation tool v0 完成

前回から継続して以下を追加・完成させた。

- **Exception label**: No Person / Tracking Lost / ID Switch / Exception の4ラベルをサブパネルで付与できるようにした。セグメント単位での付与なので、最初の10フレームは使える・後半から人が消える、といったケースも細かく記録できる。
- **Person ID**: アノテータが実際の人物IDを0〜38の範囲で付与できるようにした。右サイドパネルにビブス対応表画像を配置し、クリックで拡大表示。
- **Auto-save / Import / Export**: クリップ移動時に自動保存、Export JSONLで全件ダウンロード、Importで途中再開できるようにした。
- **annotation output format確定**: clip_id / person_id / work_segments（start_frame, end_frame, label, note） / annotated_at をREADMEに明記。

### extract_clips.pyのカメラ分割問題修正

クリップが `(track_id, cam)` 単位で分割されており、1人が複数カメラにまたがって移動する場合にクリップが途中で打ち切られていた問題を修正。

- `clips_data` のキーを `track_id` 単位に変更。
- フレームごとに `src_cam_duplicate_list[0]` から最適カメラを選んで切り出す。
- クリップフォルダ名: `track_{id}_{cam}` → `track_{id}`
- `clip_id`: `77_track_124_A1` → `77_track_124`
- manifest: `camera_id`（文字列）→ `cameras`（使用カメラのリスト）

## ミーティングフィードバック

annotation toolをデモした。システムの方向性には概ね合意を得た。

一方で、本格的なアノテーション開始前に以下の点を検討すべきという意見が出た。

- **最終的に欲しい正解データの形式を先に決める。** 作業ラベルだけでよいのか、区間・粒度・補助情報（向き・オブジェクトなど）も必要なのかを、認識モデルの設計から逆算して確定する必要がある。
- **認識モデルに何が必要かを先に考える。** 例えば、人が働きかけている方向・向き・関連オブジェクトが識別に効くなら、それもアノテーションするべき。
- **annotation schemaが決まってからツールを拡張・本格運用する。**

→ Design Dayを設けて、この設計を先に整理することにした。
